# 프로젝트 회의 주제

## (teamsquare_shoppingmole)

## 개요

- 주제
  - 이 프로젝트는 기본적으로 인터넷 쇼핑 기능을 함과 동시에, 본인의 현재 자산 상황을 통한 적절한 상품 항목 제시 및 구독 서비스로 인한 사용자 혜택 제공등의 기능을 담은 앱입니다
- 설계
  - 소비자
    - 1. 기본적으로 상품/카테고리
    - 2. "포인트" -> 상품 가리지 않고 해당 플랫폼에서 구매를 하면 할 수록 쌓이는 "포인트"
    - 3. 목표금액을 달성했을 때 알람으로 알려주는 기능.
  - 상품
    - 100만 판매 이벤트 등 알람 (감사 이벤트 - 할인쿠폰 )
    - 상품 = 구독 느낌으로 상품을 구매한 소비자들에게만 '알람' 서비스가 가도록.
    - 쿠팡와우 - 로켓배송/쿠팡잇츠 무료 배달/쿠팡 플레이 시청 가능.
- 추가 기능 ( 시간이 남는다면)
  - 판매자 기능 추가
    - 감사 알람 이벤트 등을 상품이 아닌 판매자가 직접 배포

### 그 외 나왔던 Plan

- 회의 내용
  - 1. **'쇼핑몰' >> 영수증 등, 사고 팔고에 대한 기본적인 프로젝트.**
    - 반박 : 너무 단순함
    - 개선 : 뼈대를 쇼핑몰로 지정 이 후 살을 붙이는 식으로 진행
  - **2.'저축/플래너' >> 목표 금액을 모으면 쇼핑몰 따위의 플랫폼 뷰에서 거래 기능이 해금되도록.**
    - 반박 : 피로도의 역효과, 구매가 해금 되는 방식은 사용자에게 스트레스로 다가갈수있음
    - 개선 : 구독형 멤버쉽, 포인트 적립, 목표 달성 알람과 같은 혜택 해금 형으로 교체
  - **3. ‘증권앱, 코인앱’ >> 원하는 투자율에 자동 판매, 금리율 비교 손실 및 이익률 최소 최대화.**
    - 반박 : 서비스 내 많은 사용자가 동일한 알고리즘을 설정했을 때 생기는 뱅크런 위험
    - 개선 : “포인트” 시스템으로 변환, 구매 후 리워드 포인트(사용자 기준 구매에 대한 부담 Down)
  - **4. 유저1, 유저2 -> 각각의 호스트들에 대한 구독/결제 시스템**
    - 반박 : 콘텐츠 퀄리티 유지 및 지속성 문제
    - 개선 : 서버에서 상품을 구매한 사용자들에게 이익이 되는 알람 서비스 제공
- 결정 내용
  - 쇼핑몰로 뼈대를 정한 후 개선 사항을 뼈대에 하나씩 붙여가는 형식으로 시작
  - 이 후 프로젝트의 여유가 생긴다면 추가기능 구현
